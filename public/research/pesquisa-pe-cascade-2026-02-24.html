§2. Conceito Fundamental

Definição precisa

Prediction Error Cascade é um mecanismo de inferência hierárquica onde cada camada de um sistema multi-nível avalia a magnitude do erro de predição (PE) e decide autonomamente se:

  1. RESOLVE o erro na camada atual (via ajuste de estados internos), ou
  2. ESCALA o erro para a camada acima (quando PE excede threshold local)

Este processo é fundamentado no Free Energy Principle (FEP) de Karl Friston, que postula que sistemas biológicos auto-organizados minimizam variational free energy — um bound informacional sobre surpresa.

F = EQ[ln Q(x) - ln P(ỹ, x)]

Onde:

  • F = variational free energy
  • Q(x) = approximate posterior (belief distribution sobre estados ocultos)
  • P(ỹ, x) = joint probability (generative model)
  • ỹ = generalized motion of observations (y, y', y'', ...)

Contexto histórico

Helmholtz (1860/1962): Percepção como "unconscious inference" — o cérebro infere causas de sensações via Bayesian reasoning. Seminal, mas não formalizou a hierarquia.

Rao & Ballard (1998): Predictive coding na visual cortex — erro de predição propagado bottom-up, predições enviadas top-down. Primeiro modelo computacional formal.

Friston (2003-2009): Generalizou predictive coding via FEP + hierarchical dynamical models. Paper seminal: "Predictive coding under the free-energy principle" (2009, Royal Society, 2062 citações). Introduziu generalized coordinates of motion.

Mirza et al. (2016): Active inference + scene construction — hierarchical belief-updating via expected free energy decomposition (epistemic value + instrumental value).

Verses AI (2024): PCX library — equilibrium propagation com predictive coding energy. Identificou problema crítico: energy concentration nas últimas camadas impede scaling.

Evolução do conceito

  • 1860-1998: Percepção como inferência (conceitual)
  • 1998-2009: Predictive coding computacional (visual cortex)
  • 2009-2016: FEP + hierarchical inference (generalização)
  • 2016-2024: Active inference + implementação (ação + percepção)
  • 2024-2026: Scaling problems + energy balance (Verses AI, PCX)

§3. Mecanismo

Como funciona tecnicamente

O cascade opera via hierarchical message passing entre dois tipos de unidades neurais:

1. Error Units (ξ(i)) — Superficial Pyramidal Cells

Computam precision-weighted prediction error:

ξ(i) = Π̃(i) · ε̃(i)

Onde:

  • ε̃(i) = raw prediction error (observação - predição)
  • Π̃(i) = precision matrix (inverse covariance do ruído)
2. State Units (μ̃(i)) — Deep Pyramidal Cells

Codificam conditional mean dos estados ocultos. Atualização:

μ̃̇(i) = Dμ̃(i) - ∂μF

Onde:

  • D = derivative operator (generalized coordinates)
  • ∂μF = gradient da free energy

Componentes do Sistema

Decisão de Escalação (Threshold Logic)

Regra de decisão em cada camada i:

  1. Compute PE: ε̃(i) = ỹ(i) - g̃(i)(μ̃(i))
  2. Weight by precision: ξ(i) = Π̃(i) · ε̃(i)
  3. Check magnitude:
    • Se ||ξ(i)|| < θi → RESOLVE localmente (adjust μ̃(i))
    • Se ||ξ(i)|| ≥ θi → ESCALA para i+1 (propagate ξ(i))

Threshold θi determinado por:

  • Precision Π̃(i): Camadas com alta precision (baixo ruído) têm threshold menor → resolvem mais
  • Learning rate γ: Smaller γ → slower updates → higher threshold (Verses AI finding)
  • Energy concentration: Se energia da camada i+1 >> energia da camada i, threshold efetivo aumenta (gradient vanishing)

FEP como Gating Mechanism (Luis, 24/02)

"Estímulo vai pela via de menor resistência. PE não resolvido → escala pra camada acima."

— Luis Adures, Training Session, 2026-02-24 12:00

Interpretação: O cascade é um energy minimization path. Cada camada tenta resolver PE localmente (menor custo energético). Se não consegue (PE residual alto), escalona para camada superior (que tem mais "degrees of freedom" para reconfigurar predições).

Condições de Contorno

Quando o cascade funciona:

  • Precision equilibrada entre camadas (Π̃(i) ≈ Π̃(i+1))
  • Learning rate γ pequeno (convergência lenta mas estável)
  • Energy balance (energia de camadas adjacentes na mesma ordem de magnitude)

Quando o cascade falha:

  • Energy concentration: Energia de L(n) >> L(1) → exponentially small gradients (Verses AI, 2024)
  • High learning rate γ: Test accuracy degrada, energy imbalance aumenta (PCX benchmark: γ ↑ → performance ↓)
  • Deep networks: ResNet18 com iPC: accuracy ↓ conforme depth ↑ (opposite de backprop)

Active Inference (Frontiers AI, 2020) — Expected Free Energy

Decomposição de G (expected free energy):

G(π, τ) ≤ -E[ln P(oτ)] - E[H[P(sτ|oτ)]]

Onde:

  • Left term = information gain (epistemic value — reduce uncertainty sobre estados)
  • Right term = extrinsic value (instrumental — match prior preferences)

Alternative decomposition:

G = E[Ambiguity] + E[Risk]
  • Ambiguity: Expected uncertainty of observations given states
  • Risk: KL-divergence entre expected outcomes e preferred outcomes

§5. Debates Existentes

Controvérsia 1: Scaling Problem — Fundamental ou Implementacional?

Posição A (Verses AI): "Energy concentration é problema fundamental de equilibrium propagation. Small learning rates → better performance mas exponentially small gradients quando depth ↑."

Posição B (Friston Lab): "Hierarchical precision weighting resolve scaling. Problema é lack of proper precision estimation, não FEP per se." (Friston 2018)

Evidência empírica: PCX benchmark mostra que VGG7 = backprop, ResNet18 < backprop. Sugere que scaling IS a problem, mas não prova se é fundamental.

Controvérsia 2: Precision Weighting — Computed ou Learned?

Escola 1 (Bayesian): Precision Π̃ deve ser computada via inverse covariance do ruído observado (noise statistics).

Escola 2 (Neural): Precision deve ser aprendida via neuromodulators (dopamine, acetylcholine) — "attention as precision weighting" (Feldman & Friston 2010).

Implicação prática: Anima usa precision fixa ou variável? Se fixa → não adapta a context. Se variável → quem controla?

Controvérsia 3: FEP é Falsificável?

Crítica (2024): "FEP é tão geral que explica tudo e nada. Não há experimento que possa falsificar." (Andrews 2021, Walsh et al. 2024)

Defesa (Friston): "FEP é princípio normativo, não teoria empírica. Falsificação ocorre em implementações específicas (e.g., predictive coding), não no princípio."

Stance atual: Debate filosófico não resolvido. Pragmaticamente: FEP é framework útil para derivar algoritmos (PC, active inference).

§6. Pontos de Crítica

Limitação 1: Energy Concentration (Verses AI, 2024)

[T] Fonte: Verses AI, "Benchmarking Predictive Coding Networks Made Simple" (2024)
Finding: "Energy in the last layer is orders of magnitude larger than the one in the input layer, even after performing several inference steps."
Implicação: Gradient vanishing quando depth ↑ → PC não scala como backprop.

Evidência quantitativa:

  • Small γ (learning rate) → better performance, mas large energy imbalance
  • Energy ratio R = EL(n) / EL(1) > 103 em ResNet18
  • ResNet18 com iPC: accuracy ↓ conforme layers ↑ (opposite de backprop)

Limitação 2: Computational Intractability (Andrews 2021)

[E] Fonte: Andrews (2021), "The Math is not the Territory: Navigating the Free Energy Principle"
Crítica: "Computing true posterior P(x|õ) via FEP requer marginalization sobre todos os estados possíveis — computationally intractable para high-dimensional models."
Resposta (Friston): "Variational Bayes usa approximate posterior Q(x), não true posterior. Tractability via Laplace approximation."

Limitação 3: Mock Implementation (Anima, 2026)

Observação empírica (24/02): Anima cost logs mostram $0.0000 para cascade via GLM, mas ~$0.59 para L7 responses via Claude. Isso sugere:

  • GLM não está sendo chamado (mock/placeholder), ou
  • GLM está retornando resposta instantânea sem inference (cache hit impossível em PE computation)

Implicação: PE Cascade pode estar "funcionando" (L2/L3 resolution confirmado) mas com lógica mock, não real inference.

Limitação 4: Lack of Precision Adaptation (Mondragon, 2018)

[E] Fonte: Angel Mondragon (Medium, 2018), "Hierarchical Predictive Coding"
Crítica: "Prediction errors are assigned importance, weighted, and reliability based on the environment and context. [...] complex neurotransmitter economies are implicated here, with dopamine as a major contributor."
Gap em implementações atuais: PCX, Anima, PrediRep usam precision FIXA. Não há adaptive weighting via context.

Inspiração: Mondragon (2018) — "Precision-weighted PE signals enjoy enhanced post-synaptic gain."

Implementação:

// precision-adapter.ts
class PrecisionAdapter {
  computePrecision(context: MessageContext): number {
    // High urgency → high precision → lower threshold → resolve locally
    if (context.urgency === 'critical') return 2.0;
    
    // Low confidence in priors → low precision → higher threshold → escalate
    if (context.priorConfidence < 0.3) return 0.5;
    
    // Default
    return 1.0;
  }
}
